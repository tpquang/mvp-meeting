# Coding Rules & Guidelines

## Overview

This document serves as the main entry point for all coding rules and guidelines in this project. Following these rules ensures consistent, maintainable, and high-quality code.

## Documentation Structure

- [Function Rules](./function-rules.md) - Rules for function length, complexity, and structure
- [File Rules](./file-rules.md) - Rules for file length and organization
- [Naming Conventions](./naming-conventions.md) - Naming standards for variables, functions, components, and files
- [General Guidelines](./general-guidelines.md) - General best practices and coding standards
- [TypeScript Guidelines](./typescript-guidelines.md) - TypeScript-specific rules and patterns
- [React & Next.js Guidelines](./react-nextjs-guidelines.md) - React and Next.js best practices

## Quick Reference

### Key Limits

| Rule | Limit |
|------|-------|
| Function body lines | 30 lines |
| File total lines | 300 lines |
| Function parameters | 4 parameters |
| Nesting depth | 3 levels |
| Component props | 7 props |

### Enforcement

These rules are enforced through:
- ESLint configuration
- Code review processes
- Pre-commit hooks (optional)

## Contributing

When adding new rules or modifying existing ones, please:
1. Update the relevant documentation file
2. Update ESLint configuration if applicable
3. Communicate changes to the team
